## Storage policy

Disk is cheap. Attention and floor space are not.

| Class | What belongs | Retention | Action when over policy |
|-------|--------------|-----------|-------------------------|
| **Must-keep data** | Family photos, documents, configs, backups of real systems | _____ years / generations | Expand storage or prune oldest per written rule |
| **Working media** | Current media library actually used | “Still watched / still needed” | Delete or move offsite |
| **Curated ISO / install mirror** | Current LTS + tools you rebuild from | Current + previous major only | Delete the rest |
| **Cold archive** | Rare legal/historical keepers | Offsite or powered-down shelf, inventoried | Not on the hot rack floor |
| **Collection / “just in case”** | Everything else (e.g. 100TB ISO sprawl) | **Not allowed** as a mission | Delete or offsite within 45 days of audit |

## Exit rules

| Trigger | Action |
|---------|--------|
| Unused **90 days** | Move to Decide tote |
| Still undecided by **day 120** | Automatic Exit (sell / gift / recycle) |
| No named job + powered | Power off this week; charter or kill within 30 days |
| “I might need it someday” | Not a job — Decide tote with a date |
| “For parts” without a labeled parts bin + list | Lying — Exit |

**Three exits only:** Sell · Gift · Recycle / e-waste  
No fourth category called “maybe later” on the floor.
