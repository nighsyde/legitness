# Stewardship Ritual — Biweekly Block

*The new meaning is stewardship under constraint. Reliability over novelty. Hours that fit nap windows. The stack must fit the life.*

**Default:** 45–90 minutes every other weekend.

## Ritual agenda (run in order)

Hard stop at 90 minutes. Leftover work waits for the next block — or forces a cut.

### 1. Health (10–20 min)

- [ ] Backups: last successful run for must-keep data  
- [ ] Disk / SMART / free space vs charter policy  
- [ ] UPS / power oddities  
- [ ] Anything alerting that household utility depends on  

### 2. Perimeter (5–15 min)

- [ ] Walkways clear?  
- [ ] Decide tote: still before deadline? Process any easy Exits  
- [ ] Bench: empty or only the one charter project?  
- [ ] Exit staging: schedule leave dates  

### 3. Updates that matter (10–25 min)

- [ ] Security / OS updates on Keep hosts only  
- [ ] No drive-by rearchitecting  
- [ ] Note changes on [`07-restore-one-page.md`](07-restore-one-page.md) if topology or restore steps changed  

### 4. One intentional touch (0–30 min) — optional

Either:

- Work the **single** charter project, **or**  
- Stop and leave early (a short clean ritual is a win)

Do not start a second project in leftover minutes.

### 5. Close (5 min)

- [ ] Door inventory still accurate?  
- [ ] Log the session below  
- [ ] Next block still on calendar?

## Shrink trigger checklist

If any of these are true, cut before you “organize more”:

- [ ] Two stewardship skips in a row  
- [ ] Cannot name every 24/7 host’s job in one breath  
- [ ] Decide tote past its date  
- [ ] Hours needed exceed charter monthly max  
- [ ] Family veto raised (noise/heat/safety/space)  

Action: power off non-essential → update restore doc → Exit path for what stays off.
